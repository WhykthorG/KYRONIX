# Plano de Reorganização — KYRONIX S.E.N.O (MIMO)

## Contexto

O projeto é um monorepo (workspaces: frontend, backend, shared) com alias `@` → `frontend/src/` e `@shared` → `shared/src/`. O Vite resolve os aliases no build e no dev server. A análise confirmou todos os problemas listados pelo usuário. Este plano define ações concretas em fases.

---

## Fase 3 — Consolidar hooks redundantes

### 3.1 use-mobile.jsx vs useIsMobile.js

Confirmado: dois hooks que fazem a mesma coisa:
- `hooks/use-mobile.jsx` — exporta `useIsMobile()` (shadcn/ui style, breakpoint fixo 768)
- `hooks/useIsMobile.js` — exporta `useIsMobile(breakpoint)` (parâmetro configurável, versão mais completa)

**Ação:**
1. Manter `hooks/useIsMobile.js` (mais completo, aceita breakpoint como parâmetro)
2. Atualizar todos os imports de `use-mobile` para `useIsMobile`
3. Deletar `hooks/use-mobile.jsx`

### 3.2 useWindowDrag.ts

Único arquivo `.ts` em hooks. Manter como está — é TypeScript com tipos explícitos (WindowPosition, WindowSize, WindowRect). Não há problema em misturar .ts e .js neste projeto (tsconfig.json já inclui `**/*.ts`).

## Fase 6 — Consolidar schedule-planner (opcional / médio prazo)

O schedule-planner está dividido em 3 locais:
1. `frontend/src/modules/schedule-planner/` — 6 componentes + constants + hooks + 1 service
2. `frontend/src/lib/schedulePlannerClient.js` — API client (será movido para services/ na Fase 5)
3. `shared/src/scheduling/engine.js` — Engine de geração (shared, faz sentido ficar lá)
4. `backend/src/services/schedule-planner/` — service.js + validators.js (backend, faz sentido ficar lá)

**Recomendação:** NÃO mover o `modules/schedule-planner/` agora. A consolidação já é parcial (o client vai para services/). A estrutura atual de modules/ é funcional. Futuramente, se quiser criar `features/schedule-planner/`, é uma refatoração separada.

## Riscos e Mitigações

| Risco | Mitigação |
|-------|-----------|
| Import quebrado após mover arquivo | Fazer grep completo antes de cada move; rodar build no final |
| `lib/auditLog.jsx` código morto | Confirmado: nenhum arquivo importa. Deletar diretamente |
| `lib/stores/` move quebra imports relativos | Os stores importam de `../appManifest.js` — após mover para `stores/`, o path relativo muda para `../lib/appManifest.js` |
