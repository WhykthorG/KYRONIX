## Fases de Execucao

### Fase 1: Preparacao do TCC base
1. Criar branch `develop` no repo TCC
2. Mover 122 arquivos .bak para `archive/bak-files/`
3. Deletar `TCC.rar`
4. Remover `@anthropic-ai/sdk` do package.json
5. Remover `xlsx` do frontend/package.json
6. Adicionar `exceljs` ao root e frontend package.json
7. Renomear projeto para KYRONIX S.E.N.O em todos os configs
8. Criar novo .env.example completo

### Fase 2: Modulos do MIMO → TCC
Copiar e adaptar de MIMO para TCC:
- **Paginas** (14): Internships, InternshipCompanies, TCCProjects, Laboratories, Courses, Certificates, SchoolCalendar, StudentEnrollment, AssessmentsImprovements, ExamCalendar, StudentSchedule, StudentAttendance, StudentInternship, StudentTCC
- **Componentes**: enrollment/, student/, teacher/
- **Rotas backend** (15+): admin/internships, admin/internship-companies, admin/tcc-projects, admin/laboratories, admin/courses, admin/certificates, admin/series, tcc/deliveries, internships/diary, laboratories/reservations
- **Contratos** (25): todos de shared/src/contracts/
- **Supabase**: migrations especificas dos modulos novos

### Fase 3: Backend Hibrido
1. Migrar modulos admin do TCC para MVC (controllers/models)
2. Criar controllers/ para: enrollments, schedule-planner
3. Manter flat para: grades, attendance, messages
4. Adaptar imports de todos os arquivos copiados

### Fase 4: Frontend
1. Copiar i18n/ do MIMO para TCC
2. Copiar stores/ do MIMO para TCC
3. Copiar components/desktop/ extras do MIMO
4. Atualizar appManifest.js com modulos novos
5. Atualizar routes/index.js com paginas novas
6. Atualizar App.jsx com imports corrigidos
7. Atualizar access.js com 72 permissoes do MIMO
8. Adicionar pages faltantes ao PAGE_PERMISSION_MATRIX
9. Tornar dark mode opcional no SettingsPage
10. Remover chamadas de video (Calls) do appManifest e routes
11. Remover ClientRuntimeObservability do frontend

### Fase 5: Infra e Deploy
1. Atualizar netlify.toml com included_files do MIMO
2. Manter vercel.json do TCC
3. Atualizar Dockerfile com nome KYRONIX
4. Copiar scripts/ do MIMO (security-check.mjs, etc.)
5. Copiar SECURITY_TEST_GUIDE.md

### Fase 6: Documentacao
1. Substituir README.md pelo do MIMO
2. Copiar docs/ do MIMO (26 arquivos)
3. Copiar memoria/ do MIMO
4. Copiar .mimocode/ do MIMO
5. Criar CHANGELOG.md inicial

### Fase 7: Supabase Schema
1. Analisar schema do MIMO e do TCC
2. Gerar schema unico consolidado
3. Consolidar migrations

### Fase 8: Testes
1. Manter 26 testes existentes
2. Adicionar testes para: internships, laboratories, courses, certificates, TCC projects, school calendar, student enrollment

### Fase 9: Deploy
1. Rodar `npm run build` para verificar
2. Rodar `npm run test` para verificar
3. Criar repo KYRONIX no GitHub
4. Push para main + develop
